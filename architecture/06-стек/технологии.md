# Стек технологий

| Слой | Выбор | Альтернативы | Почему |
|---|---|---|---|
| Язык | Python 3.11+ | Node.js, Go | aiogram, Pydantic |
| Бот | aiogram 3.4+ | telebot, pyrogram | FSM, middleware, асинхронность |
| HTTP | httpx 0.27+ | requests, aiohttp | Асинхронный, таймауты, современный |
| Валидация | Pydantic 2.5+ | dataclasses, marshmallow | Строгая типизация, валидация JSON |
| Конфиг | pydantic-settings | dotenv, dynaconf | Типизированные settings из .env |
| База | SQLite | PostgreSQL, Redis | Ноль зависимостей, один файл, WAL |
| AI | OpenRouter + YandexGPT | OpenAI, Anthropic | Дешевле, без VPN, российский fallback |
| Деплой | Docker + compose | K8s, bare metal | Воспроизводимость, простота |
| Логи | logging → stdout | loguru, structlog | Стандартно, совместимо с Docker |

## Что НЕ выбрали

| Технология | Почему отказались |
|---|---|
| PostgreSQL | Избыточен для MVP (< 100 пользователей). SQLite через WAL держит конкурентность. |
| Redis | SQLite покрывает FSM и кэш. Нет смысла добавлять зависимость. |
| FastAPI + Webhook | Аудит — синхронный запрос. Polling проще, webhook не даёт преимуществ. |
| Docker Swarm / K8s | Один сервер, один контейнер. Масштабирование — когда 500+ пользователей. |
| Векторные базы | Нет семантического поиска. Промты статические. |
| Next.js / веб-интерфейс | MVP — Telegram. Веб — Фаза 3. |
