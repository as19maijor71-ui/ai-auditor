# План функции: дополнение данных из экспорта (supplement flow)

## Логика (описана селлером)

1. Селлер загружает XLSX/CSV из ЛК WB/Ozon
2. Бот показывает список товаров → селлер выбирает один
3. Бот показывает:
   - Что **уже есть** в экспорте: название, бренд, цена, описание (если было)
   - Что **нужно добавить**: описание, скрины фото, характеристики, видео
   - Кнопка «Запустить аудит» — **всегда видна**
4. Селлер отправляет текст/скрины — **бот молча копит данные**
5. После каждого действия бот **редактирует ОДНО сообщение-чеклист**:
   - «✅ Описание получено. Осталось: скрины, характеристики. [Запустить аудит]»
   - «✅ Скриншот получен. Осталось: характеристики. [Запустить аудит]»
   - «✅ Всё собрано! [Запустить аудит]»
6. Селлер **сам решает** когда нажать «Запустить аудит»
7. Бот НЕ решает за селлера что «всё готово»

## Технические правила

- **ОДНО сообщение чеклиста** — всегда `edit_message_text`, никогда не создавать новые
- **Никакого debounce/asyncio задач** — прямой вызов `_update_checklist` из хендлера
- **«Message not modified» → return** — не создавать новое сообщение
- **Кнопка «Запустить аудит»** — в каждом обновлении чеклиста
- **Накопление текста** — append через `\n---\n`, обрезка до 6000 символов
- **FSM-состояние** — `supplementing_export`, данные в `accumulated_text`
