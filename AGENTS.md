# AI Аудитор WB/Ozon — навигация

Карта проекта. Читай в начале каждой сессии.

## Что это

**AI Аудитор WB/Ozon** — Telegram-бот. AI-аудитор, который анализирует карточку товара на Wildberries или Ozon по ссылке и выдаёт отчёт: что не так, что исправить, как улучшить конверсию и ранжирование.

**Слоган:** Загрузи ссылку — получи отчёт.

## Стек

- **Язык:** Python 3.11+
- **Бот:** aiogram 3.4+
- **HTTP:** httpx 0.27+
- **Валидация:** Pydantic 2.5+
- **AI:** OpenRouter (deepseek-r1) + YandexGPT fallback
- **База:** SQLite
- **Деплой:** Docker + docker-compose (polling)

## 9 правил репо

1. **НИКАКОГО КОДА до проработки плана.** Сначала idea → architecture → plans, потом код.
2. **Не деплоить без моей явной команды.**
3. **Не коммитить без явной команды.**
4. **Не пушить без подтверждения.**
5. **Не менять idea/ и architecture/ без вопроса.**
6. **Большая фича (≈1 час+) = план в plans/.** Инструкции, деплой, мелкие правки — без плана, просто делай.
7. **Строгая типизация.** Все функции с аннотациями.
8. **.env только через config.py.** Никогда не читать .env напрямую.
9. **Лимиты — до начала, не после.** Проверка бесплатных аудитов должна срабатывать ДО того как пользователь начнёт загружать данные. Если лимит исчерпан — отказ сразу, а не после 30 минут работы.

## Большая фича = план

План создаётся только для фич, которые занимают ≈1 час или более. Инструкции по деплою, мелкие исправления, справки — делаются сразу, без отдельного плана.

- Имя: `plans/YYYY-MM-DD-имя.md`
- Статусы: `[ ]` / `[WIP]` / `[x]`
- По задаче: цель / deliverable / критерий «сделано»
- В конце — итог

## Цикл из 5 ходов после фичи

1. Найди ошибки → докажи → реши
2. Что упустили → докажи
3. Это РЕАЛЬНО проблема нашего проекта?
4. Свежий взгляд после правок
5. Security review
